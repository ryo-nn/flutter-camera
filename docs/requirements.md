# flutter-camera 要件定義・設計方針

## 1. 概要

Flutter製の自撮りアプリ。撮影した写真に「パターン」として登録した自動加工(フィルター+フレーム/スタンプ合成)を適用し、Instagram・Xへ一括投稿できる。

## 2. ポジショニング・ターゲット(プロダクト戦略で確定)

**「美容・コスメ・ファッション系のナノ〜マイクロインフルエンサー(フォロワー1,000〜10万、Instagramプロアカウント保有、主に18〜34歳女性、日本国内)向けのセルフブランディング投稿インフラ」**

- 訴求メッセージ: 「撮る→いつもの加工→InstagramとXへ同時公開。数タップで完了」(美顔カメラではなく「個人向け投稿インフラ」として売る)
- Instagramプロアカウント要件はターゲットフィルターとして活用(公開前提で発信する層だけが残る)
- フィルター品質競争(SNOW/BeautyPlus)は避け、「加工の再現性(パターン)+投稿までの垂直統合」で差別化
- 非ターゲット: 鍵垢のカジュアル自撮り層、企業のSNS運用チーム

## 3. 機能要件

### 3.1 コア機能

- **撮影**: フロントカメラでの自撮り撮影(カメラ切替可)
- **自動加工**: 撮影後、事前登録した「パターン」を選ぶと以下が自動適用される
  - 画像加工: 明るさ・彩度・美肌補正などのフィルターパラメータ
  - 合成: フレーム/スタンプ画像の重ね合わせ
- **パターン管理**: パターンの作成・編集・保存・選択(プリセットとして管理)
- **SNS一括投稿**: Instagram / X へチェックボックスで選択し同時投稿
- **投稿履歴**: 各SNSごとの成功/失敗ステータス表示

### 3.2 収益化・コスト制御機能(P0)

- **課金プラン(案B: 2ティアサブスク+超過クレジット併売)**:
  - 無料: Instagram投稿は実質無制限(Metaレート上限24時間100投稿の範囲内)+X投稿 月2〜3回
  - Light 月額¥480: X投稿 月30回
  - Pro 月額¥980: X投稿 月150回+プレミアムパターン
  - 超過分: 消費型「X投稿10回パック ¥200」(無期限・リストア対応必須)
  - 月間枠は「プラン特典(月次リセット)」、購入クレジットは無期限として分離実装(Appleガイドライン3.1.1対応)
  - 3日間無料トライアル(Pro相当)+オンボーディング直後の初回同時投稿1回保証
  - ※各プランのX枠(30/150回)は暫定値。X API側の日次上限・アプリ全体レート制限(§9)の確定後に最終決定
- **X投稿本文のURL検出・ブロック**: URL付きポストは$0.200/件(通常$0.015の13.3倍)のため、投稿画面のバリデーションで本文中のURLを検出しブロックする(当面は全ブロック)
- **X残回数表示**: 「今月あと◯回」をUI表示。上限到達時はプラン案内/クレジット購入への導線を表示
- **プラン別枠制御**: Cloud Functionsでプラン別月間X投稿枠と購入クレジット残高をサーバー側でenforcement(クライアント側制御のみは不可)
- **無料枠の乱用対策**: 無料X枠の解放条件に電話番号認証(SMS)、無料X枠はデバイス単位管理、Firebase App Checkによる多重登録・エミュレータ検知

### 3.3 リテンション・オンボーディング機能(P1)

- **プリセットパターン3〜5個同梱**: 初回セッション内で「撮る→パターン適用→投稿」を完了させるオンボーディング
- **投稿履歴の成果ダッシュボード化**: 今月の投稿数・使ったパターンランキングを表示(連続投稿日数はv1.x送り。X投稿増インセンティブは設計原則として作らない)
- **公式パターンのリモート配信基盤**: Firestore経由で運営が新パターンを配信できる仕組み(初期は隔週〜月1回配信)

## 4. 技術的な制約(公式ドキュメント確認済み)

| 項目 | 内容 | 出典 |
|---|---|---|
| Instagram投稿対象 | Business/Creator(プロアカウント)必須。個人アカウントには投稿不可 | [Instagram Platform Docs](https://developers.facebook.com/docs/instagram-platform/content-publishing/) |
| Instagram投稿フロー | ①メディアコンテナ作成(`POST /media`、画像は公開URL必須) → ②`POST /media_publish` | 同上 |
| Instagram画像形式 | JPEGのみ(MPO/JPS不可) | 同上 |
| Instagramレート制限 | 24時間で100投稿まで | 同上 |
| Instagram権限 | Instagram Login方式: `instagram_business_basic` + `instagram_business_content_publish`(Facebookページ連携不要) | 同上 |
| X投稿 | サブスクなし、従量課金(通常投稿 $0.015/件、URL付き $0.200/件)。無料枠の記載なし | [X API Pricing](https://docs.x.com/x-api/getting-started/pricing) |

### 設計上の帰結

- Instagram投稿には加工後画像を一度公開URLに置く必要がある → Firebase Storageに一時アップロード→公開URL発行→Graph API取得→投稿後削除
- OAuthクライアントシークレットはモバイルアプリに同梱しない。認可コード取得はクライアント側、トークン交換はCloud Functions経由
- 「Instagram投稿無制限」という表現は使わず、「実質無制限(Metaレート上限の範囲内)」に統一する
- Instagram投稿もStorage帯域・Functions実行の変動費(概算¥0.035/投稿)が発生するため、無料層にもサーバー側のフェアユース上限(スロットル)を設ける

## 5. 確定した事業方針

- **Instagram**: プロアカウント(Business/Creator)のみ対応。連携時に判定し、非対応の場合はエラー表示+プロアカウントへの切替手順を案内するガイドを表示する。切替ガイドは単なるエラー処理ではなく、インフルエンサー志望層を取り込む獲得装置としてオンボーディングに統合する
- **X**: 投稿コストはアプリ運営側が負担。プラン別のユーザー単位投稿枠をCloud Functions側で集計・制限してコストを制御する
- **収益化**: 案B(2ティアサブスク+超過クレジット併売)を採用。初期リリースから有料プランを搭載する
- **X投稿増インセンティブの禁止**: X投稿回数を増やす方向の施策(ストリーク・ランキング等へのX投稿算入)は運営コスト増に直結するため作らない。習慣化トリガーはコスト非連動の行動(撮影・パターン加工・Instagram投稿)に置く

## 6. 技術選定

- **Flutter**: Riverpod 3.0(コード生成) + GoRouter + Freezed + Dio、Feature-First構成
- **カメラ**: [`camera`](https://pub.dev/packages/camera)(Flutter公式パッケージ)
- **画像加工/合成**: [`pro_image_editor`](https://pub.dev/packages/pro_image_editor)(フィルター・フレーム・ステッカー合成に対応)
- **OAuth**: [`flutter_appauth`](https://pub.dev/packages/flutter_appauth)(PKCE対応、認可コード取得まで)。トークン交換はバックエンドで実施
- **課金(IAP)**: [`purchases_flutter`](https://pub.dev/packages/purchases_flutter)(RevenueCat。無料枠 $2,500 MTR以内で立ち上げ、レシート検証・プラン管理を委譲)
- **バックエンド**: Firebase(Auth / Firestore / Storage / Cloud Functions v2, region: asia-northeast1)
  - Cloud Functions: OAuthトークン交換・リフレッシュ、Instagram/X API呼び出しの代行(シークレット保護・利用量集計のため)、プラン枠・クレジット残高のenforcement
  - Firebase App Check: 多重登録・不正クライアント対策
  - Remote Config(または Firestore設定ドキュメント): X投稿枠の上限値をハードコードせずサーバー側設定値とし、X API価格改定時に即日調整可能にする

## 7. 画面構成

1. スプラッシュ
2. オンボーディング(SNS連携の説明、Instagramプロアカウント要件の案内、初回投稿完了までの導線、プロアカウント切替ガイドを統合)
3. ログイン/サインアップ(Firebase Auth)
4. ホーム/撮影画面(カメラプレビュー + パターン選択カルーセル)
5. 加工プレビュー画面(パターン適用結果の確認・微調整)
6. パターン管理画面(プリセット一覧・新規作成・編集、公式配信パターンの受信表示)
7. SNS投稿画面(投稿先選択・キャプション入力、URL検出バリデーション、X残回数表示)
8. 投稿履歴/ステータス画面(成果ダッシュボード: 今月の投稿数・パターンランキング)
9. SNSアカウント連携設定画面(Instagram/X 連携・解除、Instagramプロアカウントへの切替案内)
10. ペイウォール/プラン管理画面(Light/Pro案内、3日トライアル、クレジット購入、リストア)

## 8. データモデル(概略)

- `User`: uid, displayName, connectedAccounts(instagram/x連携状態), **plan(free/light/pro), monthlyPostQuota(月次リセット枠), postCredits(購入クレジット残高・無期限)**
- `Pattern`: id, name, filterParams(brightness/saturation/smoothing等), frameAssetId, stampAssetIds, **isOfficial(公式配信フラグ), isPremium(Pro限定フラグ)**
- `Post`: id, userId, imageUrl, targets(instagram/x), targetごとのstatus, createdAt

※ 月次リセット枠と購入クレジットは別フィールドで管理(Apple 3.1.1: 購入クレジットは失効不可)

## 9. 未確定・要検討事項

- Meta App Review(`instagram_business_content_publish`等の権限審査)のリードタイム。**審査通過をリリース判定条件とする**
- Instagramへの投稿時に一時ホスティングする画像の保持期間・削除タイミングの詳細設計
- X API側の日次上限・アプリ全体レート制限の確認(プラン枠30/150回の最終決定はこの確定後)
- **X従量課金の単価適用条件の実測検証**: (a) メディア(画像)付き投稿に通常$0.015が適用されるか(本アプリは全件画像付きのため、別料金体系なら粗利計算が崩れる)、(b) URL付き$0.200の判定仕様。**検証完了をリリースおよび価格確定の前提条件とする**
- 公式パターン素材の制作単価見積もり(配信計画確定の前提タスク)
- 価格の最終確定前に、競合のApp Store/Google Play実売価格の再調査(戦略の相場観は一部出典未確認のため)

## 10. 事業リスクと対応(要点)

| リスク | 対応 |
|---|---|
| X API価格再改定(直近: 2026年2月$0.010→4月$0.015) | 投稿枠上限をサーバー側設定値化し即日調整可能に。利用規約に枠変更の可能性を明記 |
| Meta App Review審査落ち | 審査通過をリリース判定条件とする。審査中は開発モード+テストユーザーで検証 |
| Apple審査3.1.1(月次枠とクレジット) | 月次枠=「プラン特典」、購入分=「無期限消費型クレジット」として分離設計 |
| 無料枠の複数アカウント乱用 | SMS認証+デバイス単位管理+App Check |
| ストア手数料30%時の粗利縮小 | Small Business Program登録で15%を維持。「手数料30%かつ満枠粗利率50%未満」を価格改定の発動条件として事前定義 |
| 競合模倣(SNOW等) | Meta審査・ユニットエコノミクス設計・パターン資産ロックインが時間的障壁。美容系ビーチヘッドで先行密度を作る |
